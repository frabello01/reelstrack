Admin: trash icon next to each uploaded clip

Lets the admin kill a clip the creator uploaded directly from the
to-do detail page when the take is bad. Mirrors the creator-side
delete: file goes from Drive first, then the DB row, then the
counter decrements. If it was the last clip on the reel, is_done
flips back to false (so the reel returns to the Pending column).

- Backend: new admin endpoint DELETE /api/uploads/reels/:reelId/uploads/:uploadId
  (the existing public-token one stays for the creator-side flow)
- Frontend api.js: deleteReelUpload helper
- AdminClipBadge: split each clip row into a link + a trash button
  on the right, confirmation dialog, optimistic state + counter patch
  back to the parent so the Trello tab counts reflow live
